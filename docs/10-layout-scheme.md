# 页面布局方案

- 顶部
  - 左侧放置产品名称、当前租户或站点标识，用于快速确认当前监控范围。
  - 中间放置全局状态摘要，例如在线主机数、异常主机数、最近更新时间。
  - 右侧放置时间窗口选择、刷新状态、用户入口或设置入口。
  - 顶部整体保持单行或紧凑双行，不承载过多明细信息，主要用于导航、范围确认和全局控制。

- 中部
  - 左侧放置主机列表或主机分组，支持按状态、标签、关键字快速筛选。
  - 中间放置核心监控面板，展示当前选中主机或全局聚合的关键指标。
  - 右侧放置告警、维护窗口、事件流或最近变更，用于辅助判断异常来源。
  - 中部是页面的主要操作区域，应保留最大可视面积给实时状态、趋势图和关键指标卡片。
  - 视觉焦点放在中部的核心监控面板，优先展示最需要用户立即判断的内容，例如异常主机、CPU/内存/磁盘/网络指标趋势和当前健康状态。

- 底部
  - 左侧放置系统连接状态、SSE 订阅状态或数据源状态。
  - 中间放置简短的辅助信息，例如当前筛选条件、数据采样间隔、版本号。
  - 右侧放置次要操作入口，例如文档、反馈、日志下载或诊断信息。
  - 底部不放置核心操作，避免分散用户对中部监控信息的注意力。

- 视觉焦点
  - 页面视觉焦点是中部的核心监控面板。
  - 该区域应使用更高的信息密度、更清晰的层级和更强的状态颜色提示。
  - 顶部和底部应保持克制，作为上下文和辅助控制区域服务于中部内容。

- 移动端 App 设计规范
  - 主色调
    - 主色：`#2563EB`，用于主要按钮、当前选中状态、关键链接和高优先级操作。
    - 主色深色：`#1D4ED8`，用于按钮按下态、深色强调和重要状态边框。
    - 主色浅色：`#DBEAFE`，用于轻量背景、高亮区域和选中项背景。
  - 辅助色
    - 成功：`#16A34A`，用于在线、正常、成功反馈。
    - 警告：`#F59E0B`，用于性能接近阈值、待确认状态。
    - 危险：`#DC2626`，用于离线、失败、严重告警。
    - 信息：`#0891B2`，用于提示、同步中、辅助说明。
    - 页面背景：`#F8FAFC`，用于 App 主背景。
    - 卡片背景：`#FFFFFF`，用于内容容器和指标卡片。
    - 主文本：`#0F172A`，用于标题和关键数值。
    - 次文本：`#475569`，用于正文和辅助信息。
    - 弱文本：`#94A3B8`，用于时间、占位提示和低优先级说明。
    - 分割线：`#E2E8F0`，用于列表分隔、卡片边界和表单边线。
  - 字体层级
    - 大标题：字号 `24px`，行高 `32px`，字重 `700`，用于页面主标题和关键详情页标题。
    - 标题：字号 `20px`，行高 `28px`，字重 `700`，用于模块标题和一级信息区。
    - 副标题：字号 `16px`，行高 `24px`，字重 `600`，用于卡片标题、列表主信息和分组标题。
    - 正文：字号 `14px`，行高 `22px`，字重 `400`，用于说明文本、列表描述和表单内容。
    - 辅助文字：字号 `12px`，行高 `18px`，字重 `400`，用于时间、标签、状态说明和次要元信息。
    - 指标数值：字号 `28px`，行高 `36px`，字重 `700`，用于 CPU、内存、磁盘等核心数据。
  - 按钮
    - 主要按钮高度：`44px`，圆角 `12px`。
    - 次要按钮高度：`40px`，圆角 `10px`。
    - 图标按钮尺寸：`40px` x `40px`，圆角 `10px`。
    - 主要按钮阴影：`0 8px 20px rgba(37, 99, 235, 0.24)`。
    - 普通卡片按钮阴影：`0 4px 12px rgba(15, 23, 42, 0.08)`。
    - 按下态阴影：`0 2px 6px rgba(15, 23, 42, 0.12)`。
    - 禁用态阴影：`none`，背景使用 `#E2E8F0`，文字使用 `#94A3B8`。
